# skill-optimizer

SKILL.mdを「動的eval → フォーマット準拠 → 静的品質スコア」の3層で評価し、承認ゲートを経て改善を実装するスキル。

## 使い方

```
「スキルを改善して」「スキルをレビューして」「Sグレードにして」
```

引数なしで呼ぶと対象スキルの選択を促される。パスを指定することも可能。

### フロー

```
Phase 1（評価）
  Step 1: 対象スキルと目標グレードの確定（デフォルト: Sグレード）
  Step 2: 動的eval — 新規subagentがSKILL.mdを実行して不明瞭点・裁量補完を収集
  Step 3: フォーマット準拠チェック
  Step 4: 静的品質スコアリング（5軸100点）
  Step 5: 評価レポート提示 → ユーザー承認ゲート ← ここで確認

Phase 2（実装）
  Step 6: 改善の実装（動的eval指摘を最優先）
  Step 7: 再評価
  Step 8: 完了レポート保存
```

### 評価の優先順位

動的eval（実際に動くか）> フォーマット準拠（発火・読み込みに影響）> 静的スコア（改善方向の指針）

スコアと動的evalが矛盾する場合は動的evalを優先する。

### スコアの読み方

評価レポートにはグレードに加えて「妥当なA」「改善余地あるA」のいずれかが明示される。

- **妥当なX**: 残りギャップを埋める改善がスコア最適化になる。現状で十分
- **改善余地あるX**: 実用的な改善が存在し、実施すれば自然に上のグレードに達する

## 出力先

完了レポートは `reports/skill-optimizer/{YYYY-MM-DD}/{スキル名}.md` に保存される。
