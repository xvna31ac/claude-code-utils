# 言語スペシャリストサブエージェント

言語スペシャリストは、特定のプログラミング言語とそのエコシステムの専門的なガイドです。これらのサブエージェントは、言語の慣用句、ベストプラクティス、パフォーマンス最適化技術、フレームワークの専門知識に関する深い知識を提供します。モダンなWebフレームワーク、システムプログラミング言語、エンタープライズプラットフォームのいずれに取り組んでいる場合でも、これらのスペシャリストは慣用的で効率的かつ保守性の高いコードを書けるようサポートします。

## 言語スペシャリストの使い方

以下の場合にこれらのサブエージェントを使用してください：
- **言語固有の機能と高度なパターンの習得**
- **言語固有の技術によるパフォーマンス最適化**
- **プロダクションアプリケーションのフレームワークベストプラクティスの実装**
- **既存のコードベースの移行または近代化**
- **専門的なガイダンスによる言語固有の課題の解決**
- **言語の高度なパターンとイディオムの習得**
- **フレームワーク固有のアプリケーションの確実な構築**

## はじめかた

1. **技術スタックを特定** して適切なスペシャリストを選択する
2. **プロジェクトコンテキストを説明** — 既存のコードと制約を含めて
3. **目標を明示** — 学習、最適化、実装
4. **関連コードを共有** — コンテキストを意識した支援のために
5. **スペシャリストのガイダンスに従う** — ベストプラクティスのために

## ベストプラクティス

- **言語の慣用句を使う：** 各スペシャリストはコードを書く慣用的な方法を知っている
- **エコシステムツールを活用：** スペシャリストはエコシステム全体を理解している
- **フレームワーク慣例に従う：** 各フレームワークには独自のベストプラクティスがある
- **パフォーマンスを早期に考慮：** 言語固有の最適化は重要
- **保守性を考える：** 将来の開発者が理解できるコードを書く
