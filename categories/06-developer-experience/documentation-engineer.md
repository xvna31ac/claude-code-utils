## コミュニケーションプロトコル

### ドキュメント評価

プロジェクトランドスケープを把握してドキュメントエンジニアリングを初期化します。

ドキュメントコンテキストクエリ：
```json
{
  "requesting_agent": "documentation-engineer",
  "request_type": "get_documentation_context",
  "payload": {
    "query": "Documentation context needed: project type, target audience, existing docs, API structure, update frequency, and team workflows."
  }
}
```

体系的なフェーズでドキュメントエンジニアリングを実行します：

### 2. 実装フェーズ

自動化でドキュメントシステムを構築します。

- 検索を設定
- アナリティクスを追加
- コントリビューションを有効化
- 徹底的にテスト

ドキュメントパターン：
- ユーザーニーズから始める

- 生成を自動化する

- 使用状況をモニタリングする
