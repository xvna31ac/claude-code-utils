# 開発者体験サブエージェント

開発者体験サブエージェントは、開発をより速く・簡単・楽しくすることに特化した生産性向上の専門家です。コードのリファクタリングからドキュメント作成、ビルド最適化からGitワークフローまで、あらゆる領域をカバーします。開発プロセスの摩擦を取り除き、繰り返しタスクを自動化し、より良いツールとプラクティスでチームが効率的に働けるよう支援します。

## 開発者体験サブエージェントを使うタイミング

以下のような場合に使用します：
- 保守性向上のための**レガシーコードのリファクタリング**
- より速い開発のための**ビルドシステムの最適化**
- **開発者ツールとCLIアプリケーションの作成**
- 開発者に愛される**技術ドキュメントの作成**
- **依存関係の管理**とパッケージ更新
- **Gitワークフロー**とブランチ戦略の効率化
- 最新プラクティスへの**コードベースのモダナイズ**
- チーム全体の**開発者生産性の向上**

## 利用可能なサブエージェント

### [**build-engineer**](build-engineer.md) - ビルドシステムスペシャリスト
コンパイルとバンドルを高速化するビルド最適化エキスパート。様々なビルドツール、最適化技術、キャッシュ戦略をマスターしています。ビルド時間を分単位から秒単位に短縮します。

**使用タイミング：** ビルド時間の最適化、ビルドツールの設定、ビルドキャッシングの実装、モノレポビルドの設定、ビルド問題のトラブルシューティング時。

### [**cli-developer**](cli-developer.md) - コマンドラインツール・自動化スペシャリスト
開発者とオペレーター向けの直感的・効率的なコマンドラインツールを構築するシニアCLIエンジニア。引数パース、インタラクティブプロンプト、ターミナルUX、クロスプラットフォーム互換性を専門とし、スクリプトフレンドリーなインターフェースと既存ワークフローへのスムーズな統合に注力しています。

**使用タイミング：** 内部ツール・DevOps/運用CLI・PowerShell/Bashラッパーの設計・リファクタリング、またはパイプラインで発見しやすく・人間工学的で・自動化しやすいコマンドライン体験が必要な時。

### [**dependency-manager**](dependency-manager.md) - パッケージ・依存関係スペシャリスト
複雑なパッケージエコシステムを管理する依存関係エキスパート。バージョン解決、セキュリティ更新、依存関係最適化をマスターしています。依存関係を安全かつ最新の状態に保ちながら壊さない工夫をします。

**使用タイミング：** 依存関係の管理、バージョン競合の解決、セキュリティ更新の実装、パッケージサイズの最適化、依存関係自動化の設定時。

### [**documentation-engineer**](documentation-engineer.md) - 技術ドキュメントエキスパート
明確で包括的な技術ドキュメントを作成するドキュメントスペシャリスト。APIドキュメント、チュートリアル、開発者ガイドをマスターしています。優れたドキュメントで複雑なシステムを理解しやすくします。

**使用タイミング：** APIドキュメントの作成、開発者ガイドの作成、ドキュメントサイトの構築、既存ドキュメントの改善、ドキュメントワークフローの設定時。

### [**dx-optimizer**](dx-optimizer.md) - 開発者体験最適化スペシャリスト
開発者の摩擦を特定・排除するDXエキスパート。ワークフロー、ツール、プロセスを分析して生産性を向上させます。開発を楽に・楽しくします。

**使用タイミング：** 開発者ワークフローの改善、生産性ボトルネックの分析、開発者ツールの選定、開発環境の最適化、開発者体験の測定時。

### [**git-workflow-manager**](git-workflow-manager.md) - Gitワークフロー・ブランチ戦略エキスパート
効率的なバージョン管理ワークフローを設計するGitスペシャリスト。ブランチ戦略、マージ競合解決、Git自動化をマスターしています。Gitベストプラクティスでスムーズなコラボレーションを実現します。

**使用タイミング：** Gitワークフローの設計、ブランチ戦略の実装、複雑なマージの解決、Gitプロセスの自動化、チームへのGit研修時。

### [**legacy-modernizer**](legacy-modernizer.md) - レガシーコードモダナイゼーションスペシャリスト
古いコードベースに新しい命を吹き込むモダナイゼーションエキスパート。段階的リファクタリング、依存関係更新、アーキテクチャ改善をマスターしています。機能を壊すことなくレガシーコードを変革します。

**使用タイミング：** レガシーアプリケーションのモダナイズ、リファクタリング戦略の計画、古いフレームワークの更新、新技術への移行、コード保守性の向上時。

### [**mcp-developer**](mcp-developer.md) - Model Context Protocolスペシャリスト
AIシステムと外部ツール・データソースを接続するサーバーとクライアントを構築するMCPエキスパート。プロトコル仕様、SDK実装、本番対応統合をマスターしています。AIと外部サービスのシームレスな橋渡しを作ります。

**使用タイミング：** MCPサーバーの構築、AIツール統合の作成、Model Context Protocolクライアントの実装、AIシステムと外部APIの接続、外部データソースを使ったAIアプリケーションの開発時。

### [**powershell-module-architect**](powershell-module-architect.md) - PowerShellモジュール・プロファイルアーキテクチャエキスパート
アドホックなスクリプトをクリーンで再利用可能なモジュールと高速ロードプロファイルに変換するPowerShellアーキテクチャスペシャリスト。明確なパブリック/プライベート関数境界、堅牢なパラメーター設計、DRYヘルパーライブラリ、Windows PowerShell 5.1とPowerShell 7+のクロスバージョン互換性に注力します。

**使用タイミング：** PowerShellモジュールの構造化・リファクタリング、遅いプロファイルのスリム化、関数/パラメーター規約の設計、システム管理者とヘルプデスク向けの共有インフラツールの整理時。

### [**powershell-ui-architect**](powershell-ui-architect.md) - PowerShell GUIおよびTUIスペシャリスト
PowerShellベースのツール用のUI・UXアーキテクト。自動化モジュールの上にWinForms、WPF、Metroスタイルダッシュボード（MahApps.Metro/Elysium）、ターミナルUIを設計します。保守性を犠牲にせずに再利用可能なPowerShell/.NETロジックの上にクリーンなインターフェースをレイヤリングすることに注力します。

**使用タイミング：** PowerShellツールのグラフィカルまたはターミナルUIが必要な時、WinForms/WPF/TUI/Metroアプローチから選択が必要な時、または既存のPowerShellモジュールとスクリプトに合わせてXAMLとイベントハンドラーを構造化する際。

### [**refactoring-specialist**](refactoring-specialist.md) - コードリファクタリングエキスパート
動作を変えることなくコード構造を改善するリファクタリングマスター。デザインパターン、コードの臭い、安全なリファクタリング技術のエキスパート。コードをよりクリーンで保守しやすくします。

**使用タイミング：** 複雑なコードのリファクタリング、コードの臭いの排除、デザインパターンの実装、コード構造の改善、新機能向けのコード準備時。

### [**slack-expert**](slack-expert.md) - Slackプラットフォーム・@slack/boltスペシャリスト
@slack/bolt、Slack Web API、Events API、Block Kit UI、OAuthフローに深い専門知識を持つエリートSlackプラットフォームエキスパート。レート制限、セキュリティ、最新機能のベストプラクティスで堅牢なSlack統合を構築します。

**使用タイミング：** Slackボットの構築、スラッシュコマンドの実装、Block Kitインターフェースの作成、Slackコードのレビュー、OAuthフローの設定、Slack Events APIとの統合時。

### [**tooling-engineer**](tooling-engineer.md) - 開発者ツーリングスペシャリスト
開発者ツールを構築・統合するツーリングエキスパート。IDE設定、リンター、フォーマッター、カスタムツーリングをマスターしています。生産性を向上させる開発環境を作成します。

**使用タイミング：** 開発ツールの設定、カスタムツーリングの作成、IDEの設定、コード品質ツールの実装、開発者プラットフォームの構築時。

- パッケージ更新は **dependency-manager**
- 新しいドキュメントは **documentation-engineer**

**開発者生産性：**
- ワークフロー分析は **dx-optimizer**
- ツール設定は **tooling-engineer**
- ビルド最適化は **build-engineer**
- バージョン管理は **git-workflow-manager**

**ツール開発：**
- コマンドラインツールは **cli-developer**
- IDE統合は **tooling-engineer**
- ツールドキュメントは **documentation-engineer**
- ツールパッケージングは **build-engineer**

**コード品質：**
- コード構造は **refactoring-specialist**

## はじめ方

1. 開発プロセスの**問題点を特定する**
2. 改善のために**関連スペシャリストを選ぶ**
3. ツールとワークフローの**現状を分析する**
4. **段階的に改善を実施する**
5. 開発者生産性への**影響を測定する**

## ベストプラクティス

- **繰り返しタスクを自動化する：** 節約された時間は複利で効く
- **すべてをドキュメント化する：** 将来の開発者が感謝する
- **段階的な改善：** 小さな変化が積み重なる
- **影響を測定する：** 生産性向上を追跡する
- **ツールの標準化：** 一貫性が摩擦を減らす
- **開発者のフィードバック：** ユーザーの声を聞く
- **継続的改善：** DXは「完了」することがない
- **知識を共有する：** ベストプラクティスを広める

開発者体験スペシャリストを選んで、開発を喜びにしましょう！
