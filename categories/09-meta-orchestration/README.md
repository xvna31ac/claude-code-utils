# メタ・オーケストレーションサブエージェント

メタ・オーケストレーションサブエージェントは、複雑なマルチエージェントワークフローを管理しAIシステムのパフォーマンスを最適化する指揮者・調整役です。これらのスペシャリストはメタレベルで卓越した能力を発揮し、他のエージェントのオーケストレーション、コンテキスト管理、タスク配分、複数のAIシステム間のスムーズな協調を実現します。混沌をシンフォニーに変え、複雑なAIシステムを調和して機能させます。

## メタ・オーケストレーションサブエージェントを使用するタイミング

以下が必要な場合にこれらのサブエージェントを使用します：
- 複雑なタスクのために**複数のエージェントを調整**する
- 会話全体で**コンテキストの使用を最適化**する
- スペシャリスト間で**タスクを効率的に配分**する
- マルチエージェントシステムで**エラーを適切に処理**する
- 様々なソースから**知識を統合**する
- AIワークフローの**パフォーマンスを監視**する
- 複数のステップを含む**複雑なワークフローを設計**する
- チーム全体で**AI運用をスケール**する

## 利用可能なサブエージェント

### [**agent-organizer**](agent-organizer.md) - マルチエージェントコーディネーター
複雑なマルチエージェントコラボレーションを管理するオーケストレーションエキスパート。タスク分解、エージェント選択、結果統合を習得しています。複雑な問題を調整されたソリューションに変えます。

**使用タイミング：** 複数のエージェントの調整、複雑なタスクの分解、エージェントの依存関係の管理、結果の統合、またはエージェントワークフローの設計が必要な場合。

### [**context-manager**](context-manager.md) - コンテキスト最適化エキスパート
AI会話の効率を最大化するコンテキストスペシャリスト。コンテキストウィンドウ、情報の優先度付け、メモリ管理のエキスパートです。限られたコンテキストスペースの最適利用を確保します。

**使用タイミング：** 長い会話の最適化、コンテキストウィンドウの管理、情報の優先度付け、メモリシステムの実装、またはコンテキストオーバーフローの処理が必要な場合。

### [**error-coordinator**](error-coordinator.md) - エラー処理・リカバリースペシャリスト
優雅な障害リカバリーを確保するエラー処理エキスパート。エラーパターン、フォールバック戦略、システムレジリエンスを習得しています。障害があってもマルチエージェントシステムをスムーズに稼働させます。

**使用タイミング：** エラー処理の実装、リカバリー戦略の設計、カスケード障害の管理、システム健全性の監視、またはレジリエントなワークフローの構築が必要な場合。

### [**it-ops-orchestrator**](it-ops-orchestrator.md) - PowerShell/.NETエコシステム向けITオペレーションメタオーケストレーター
曖昧なインフラ・オペレーションタスクを適切なスペシャリストエージェントにルーティングするメタオーケストレーター。PowerShellと.NETベースのワークフローを優先します。Windowsインフラ、Azure、M365、PowerShell言語エキスパートの役割を理解し、エンドツーエンドのソリューションを提供するために調整します。

**使用タイミング：** タスクがITオペレーションやインフラ自動化の雰囲気があるが複数の領域（AD、DNS/DHCP、Azure、M365、PowerShellモジュール）にまたがる場合。このオーケストレーターはPowerShellをデフォルト実装言語として最適なサブエージェント（windows-infra-admin、azure-infra-engineer、m365-admin、powershell-5.1-expert、powershell-7-expert、powershell-module-architectなど）を選択・調整します。

### [**knowledge-synthesizer**](knowledge-synthesizer.md) - 知識集約エキスパート
複数のソースからの情報を組み合わせる知識統合スペシャリスト。情報融合、競合解決、インサイト生成のエキスパートです。多様なインプットから一貫した知識を創出します。

**使用タイミング：** 複数の視点の組み合わせ、競合情報の解決、包括的なレポートの生成、ナレッジベースの構築、またはリサーチの統合が必要な場合。

### [**multi-agent-coordinator**](multi-agent-coordinator.md) - 高度なマルチエージェントオーケストレーション
複雑なエージェントエコシステムを処理する高度なオーケストレーションエキスパート。並列処理、依存関係管理、分散ワークフローを習得しています。AIオペレーションをエンタープライズレベルにスケールします。

**使用タイミング：** 大規模なエージェントシステムの構築、並列ワークフローの実装、エージェントエコシステムの管理、分散タスクの調整、またはスループットの最適化が必要な場合。

### [**performance-monitor**](performance-monitor.md) - エージェントパフォーマンス最適化
エージェントシステムを監視・最適化するパフォーマンススペシャリスト。メトリクス、ボトルネック分析、最適化戦略のエキスパートです。すべてのエージェントのピークパフォーマンスを確保します。

**使用タイミング：** エージェントパフォーマンスの監視、ボトルネックの特定、ワークフローの最適化、メトリクスの実装、またはシステム効率の改善が必要な場合。

### [**task-distributor**](task-distributor.md) - タスク配分スペシャリスト
エージェント間の作業配分を最適化するタスク配分エキスパート。負荷分散、機能マッチング、優先度スケジューリングを習得しています。利用可能なすべてのエージェントの効率的な使用を確保します。

**使用タイミング：** エージェント間のタスク配分、負荷分散の実装、タスクキューの最適化、優先度の管理、またはエージェント作業のスケジューリングが必要な場合。

### [**taskade**](https://github.com/taskade/mcp) - 自律エージェントとMCP統合を備えたAI搭載ワークスペース
自律エージェント、リアルタイムコラボレーション、ワークフロー自動化を特徴とするAI搭載ワークスペース。Claude CodeとのMCP統合用サーバーを提供し、開発環境から直接タスク管理、プロジェクトオーケストレーション、マルチエージェントワークフローを実現します。

**使用タイミング：** AIエージェントによるタスクとプロジェクトの管理、チーム全体のワークフロー自動化、マルチエージェントコラボレーションのオーケストレーション、またはMCP経由でAI搭載プロジェクト管理をClaude Codeに統合する場合。Website: [taskade.com](https://www.taskade.com)

### [**workflow-orchestrator**](workflow-orchestrator.md) - 複雑なワークフロー自動化
洗練されたAIワークフローを設計・実行するワークフロースペシャリスト。ワークフローパターン、状態管理、プロセス自動化のエキスパートです。複雑なプロセスをスムーズな運用に変えます。

**使用タイミング：** 複雑なワークフローの設計、プロセス自動化の実装、ワークフロー状態の管理、長時間実行プロセスの処理、またはワークフローエンジンの構築が必要な場合。

## 一般的なオーケストレーションパターン

**複雑な問題解決：**
- タスク分解は **agent-organizer**
- 作業配分は **task-distributor**
- 結果組み合わせは **knowledge-synthesizer**
- 障害処理は **error-coordinator**

**大規模オペレーション：**
- エコシステム管理は **multi-agent-coordinator**
- 最適化は **performance-monitor**
- プロセス自動化は **workflow-orchestrator**
- 効率化は **context-manager**

**ワークフロー自動化：**
- プロセス設計は **workflow-orchestrator**
- 作業配分は **task-distributor**
- レジリエンスは **error-coordinator**
- 最適化は **performance-monitor**

**知識管理：**
- 情報融合は **knowledge-synthesizer**
- メモリ最適化は **context-manager**
- リサーチ調整は **agent-organizer**
- 知識ワークフローは **workflow-orchestrator**

## はじめに

1. **ワークフローをマッピング**し複雑さを特定する
2. **オーケストレーション戦略を選択**しニーズに基づいて
3. **エージェントの相互作用**と依存関係を設計する
4. **最初からモニタリングを実装**する
5. **パフォーマンスに基づいて反復・最適化**する

## ベストプラクティス

- **シンプルから始める：** 段階的に複雑さを構築する
- **すべてを監視する：** 可視性が問題を防ぐ
- **優雅に障害を処理する：** エラーを予期し計画する
- **コンテキスト使用を最適化する：** コンテキストは貴重
- **ワークフローをドキュメント化する：** 複雑なシステムには明確さが必要
- **スケールでテストする：** 小さなテストはオーケストレーション問題を見逃す
- **ワークフローをバージョン管理する：** 時間とともに変化を追跡する
- **インパクトを測定する：** オーケストレーションの利点を定量化する
