# インフラサブエージェント

インフラサブエージェントは、モダンなインフラの構築、デプロイ、維持を専門とするDevOpsとクラウドコンピューティングのエキスパートです。CI/CDパイプラインからクラウドアーキテクチャ、コンテナオーケストレーションからデータベース管理まで、あらゆることを担います。アプリケーションが信頼性高く動作し、効率的にスケールし、あらゆる環境でシームレスにデプロイされることを保証します。

## インフラサブエージェントを使用する場面

以下のニーズがある場合に使用します：
- **クラウドアーキテクチャ設計** - スケーラビリティと信頼性のため
- **CI/CDパイプライン実装** - 自動化されたデプロイメントのため
- **コンテナオーケストレーション** - KubernetesとDockerで
- **コードとしてのインフラ管理** - モダンツールで
- **データベースパフォーマンス最適化** - と管理
- **モニタリングとオブザーバビリティ** - システムのセットアップ
- **インシデント対応** - 高可用性の確保
- **インフラセキュリティ** - ベストプラクティスの実装

## 利用可能なサブエージェント

### [**azure-infra-engineer**](azure-infra-engineer.md) - Azureクラウドインフラと自動化スペシャリスト
Azureリソース設計、仮想ネットワーク、ID統合、PowerShell・Bicep・Azモジュールを使用したIaCパターンの専門家。

**使用場面：** Azureの環境設計、安全なリソースデプロイ、M365との統合、またはAzureサービスとハイブリッドIDの自動化スクリプト作成。

### [**cloud-architect**](cloud-architect.md) - AWS/GCP/Azureスペシャリスト
スケーラブルでコスト効果の高いクラウドソリューションを設計するマルチクラウドエキスパート。クラウドネイティブアーキテクチャ、サーバーレスパターン、クラウドマイグレーション戦略のマスター。主要クラウドプロバイダー間での最適なリソース使用を確保します。

**使用場面：** クラウドアーキテクチャの設計、クラウドへの移行、クラウドコストの最適化、マルチクラウド戦略の実装、またはクラウドサービスの選択。

### [**database-administrator**](database-administrator.md) - データベース管理エキスパート
大規模なリレーショナルとNoSQLデータベースを管理するデータベーススペシャリスト。パフォーマンスチューニング、レプリケーション、バックアップ戦略、高可用性の専門家。データ整合性と最適なデータベースパフォーマンスを確保します。

**使用場面：** データベースのセットアップ、クエリパフォーマンスの最適化、バックアップ戦略の実装、データベーススキーマの設計、またはデータベース問題のトラブルシューティング。

### [**docker-expert**](docker-expert.md) - Dockerコンテナ化と最適化スペシャリスト
本番グレードのDockerfile、マルチステージビルド、セキュリティ強化、コンテナパフォーマンス最適化の専門家。イメージサイズ削減、ビルドキャッシング、エンタープライズデプロイメントパターンのマスター。

**使用場面：** 最適化されたDockerfileの構築、イメージサイズの削減、マルチステージビルドの実装、コンテナイメージのセキュリティ強化、Docker Composeの設定、またはCI/CDパイプラインへのコンテナビルドの統合。

### [**deployment-engineer**](deployment-engineer.md) - デプロイメント自動化スペシャリスト
環境間のアプリケーションリリースを自動化するデプロイメントエキスパート。ブルーグリーンデプロイ、カナリアリリース、ロールバック戦略のマスター。確信を持ってゼロダウンタイムデプロイを実現します。

**使用場面：** デプロイメントパイプラインのセットアップ、リリース戦略の実装、デプロイメントの自動化、環境管理、またはデプロイメントの信頼性確保。

### [**devops-engineer**](devops-engineer.md) - CI/CDと自動化エキスパート
開発と運用を橋渡しするDevOpsプラクティショナー。CI/CDパイプライン、自動化ツール、DevOpsカルチャーの専門家。安定性とセキュリティを維持しながらデリバリーを加速します。

**使用場面：** CI/CDパイプラインの構築、ワークフローの自動化、DevOpsプラクティスの実装、開発環境のセットアップ、またはデプロイメント速度の向上。

### [**devops-incident-responder**](devops-incident-responder.md) - DevOpsインシデント管理
DevOps環境向けのインシデント対応スペシャリスト。トラブルシューティング、根本原因分析、インシデント管理のマスター。体系的なアプローチでダウンタイムを最小化し、将来のインシデントを防止します。

**使用場面：** 本番インシデントへの対応、インシデント管理プロセスのセットアップ、根本原因分析の実施、またはインシデント防止策の実装。

### [**incident-responder**](incident-responder.md) - システムインシデント対応エキスパート
システム障害と緊急事態を担当するクリティカルインシデントスペシャリスト。迅速な診断、リカバリー手順、ポストモーテム分析の専門家。失敗から学びながら迅速にサービスを復元します。

**使用場面：** クリティカルインシデントの管理、インシデント対応計画の策定、ポストモーテムの実施、またはインシデント対応チームのトレーニング。

### [**kubernetes-specialist**](kubernetes-specialist.md) - コンテナオーケストレーションマスター
スケールでコンテナ化アプリケーションを管理するKubernetesエキスパート。クラスター設計、ワークロード最適化、Kubernetesエコシステムツールのマスター。本番環境での信頼性の高いコンテナオーケストレーションを確保します。

**使用場面：** Kubernetesへのデプロイ、クラスターアーキテクチャの設計、ワークロードの最適化、サービスメッシュの実装、またはKubernetes問題のトラブルシューティング。

### [**network-engineer**](network-engineer.md) - ネットワークインフラスペシャリスト
安全でパフォーマンスの高いネットワークを設計するネットワークアーキテクチャエキスパート。SDN、ロードバランシング、ネットワークセキュリティのマスター。信頼性の高い接続と最適なネットワークパフォーマンスを確保します。

**使用場面：** ネットワークアーキテクチャの設計、ロードバランサーの実装、VPNのセットアップ、ネットワークパフォーマンスの最適化、または接続のトラブルシューティング。

### [**platform-engineer**](platform-engineer.md) - プラットフォームアーキテクチャエキスパート
内部開発者プラットフォームを構築するプラットフォームスペシャリスト。セルフサービスインフラ、ゴールデンパス、プラットフォーム抽象化を作成します。ガバナンスを維持しながら開発者を強化します。

**使用場面：** 内部プラットフォームの構築、開発者ポータルの作成、プラットフォームエンジニアリングの実装、インフラの標準化、または開発者生産性の向上。

### [**security-engineer**](security-engineer.md) - インフラセキュリティスペシャリスト
インフラとアプリケーションを保護するセキュリティエキスパート。セキュリティ強化、コンプライアンス、脅威防止のマスター。全レイヤーにわたる多層防御戦略を実装します。

**使用場面：** インフラのセキュリティ確保、セキュリティポリシーの実装、コンプライアンスの達成、セキュリティ監査の実施、またはセキュリティインシデントへの対応。

### [**sre-engineer**](sre-engineer.md) - サイト信頼性エンジニアリングエキスパート
エンジニアリングを通じてシステム信頼性を確保するSREプラクティショナー。SLI/SLO、エラーバジェット、カオスエンジニアリングのマスター。機能速度とシステム安定性のバランスを取ります。

**使用場面：** SREプラクティスの実装、SLOの定義、モニタリングのセットアップ、カオスエンジニアリングの実施、またはシステム信頼性の向上。

### [**terraform-engineer**](terraform-engineer.md) - コードとしてのインフラエキスパート
インフラ自動化にTerraformを使用するIaCスペシャリスト。モジュール設計、状態管理、マルチ環境デプロイメントのマスター。インフラの一貫性と再現性を確保します。

**使用場面：** Terraformコードの記述、IaCアーキテクチャの設計、Terraform状態の管理、再利用可能なモジュールの作成、またはインフラプロビジョニングの自動化。

### [**terragrunt-expert**](terragrunt-expert.md) - TerragruntオーケストレーションとDRY IaCスペシャリスト
大規模でOpenTofu/TerraformインフラをオーケストレーションするシニアTerragruntエキスパート。スタックアーキテクチャ、ユニットコンポジション、依存関係管理、DRY設定パターンのマスター。コードの再利用と保守性に重点を置いたエンタープライズグレードのインフラ自動化を確保します。

**使用場面：** TerragruntでTerraformモジュールをオーケストレーション、環境間でのDRY設定の実装、複雑な依存関係グラフの管理、マルチアカウント/マルチリージョンインフラの設計、またはモノリシックTerraformからモジュール型Terragruntスタックへの移行。

### [**windows-infra-admin**](windows-infra-admin.md) - WindowsインフラとActive Directory自動化エキスパート
PowerShellを使用したAD、DNS、DHCP、GPO、サーバー設定、ドメインサービスの自動化の深い専門知識。安全な変更ワークフロー、べき等操作、エンタープライズグレードの運用パターンに注力。

**使用場面：** ドメインインフラの管理、ADオブジェクトの変更、DNS/DHCPレコードの更新、GPOタスクの自動化、またはエンタープライズ環境でのサーバーレベルの自動化。

## はじめ方

1. **インフラニーズと現在の課題を評価する**
2. **要件に基づいて適切なスペシャリストを選択する**
3. **環境と制約に関するコンテキストを提供する**
4. **該当する場合は既存の設定を共有する**
5. **スペシャリストのベストプラクティスの推奨事項に従う**

## ベストプラクティス

- **アーキテクチャから始める：** 実装前に設計する
- **すべてを自動化する：** 手動プロセスはスケールしない
- **セキュリティファースト：** すべてのレイヤーにセキュリティを組み込む
- **プロアクティブにモニタリング：** オブザーバビリティがインシデントを防ぐ
- **徹底的にドキュメント化：** 将来の自分が感謝する
- **インフラをテスト：** インフラコードもテストが必要
- **障害に備える：** 回復力のために設計する
- **継続的に反復する：** インフラはニーズとともに進化する

インフラスペシャリストを選択して、今日から信頼性の高いシステムを構築しましょう！
