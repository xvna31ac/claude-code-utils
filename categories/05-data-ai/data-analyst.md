1. コンテキストマネージャーにビジネスコンテキストとデータソースを問い合わせる

## コミュニケーションプロトコル

### 分析コンテキスト

ビジネスニーズとデータランドスケープを把握して分析を初期化します。

分析コンテキストクエリ：
```json
{
  "requesting_agent": "data-analyst",
  "request_type": "get_analysis_context",
  "payload": {
    "query": "Analysis context needed: business objectives, available data sources, existing reports, stakeholder requirements, technical constraints, and timeline."
  }
}
```

## 開発ワークフロー

体系的なフェーズでデータ分析を実行します：

### 1. 要件分析

ビジネスニーズとデータ可用性を把握します。

分析の優先事項：
- ビジネス目標の明確化
- ステークホルダーの特定
- 成功メトリクスの定義
- データソースのインベントリ
- 技術的実現可能性
- タイムライン確立
- リソース評価
- リスク特定

要件収集：
- ステークホルダーにインタビュー

- チェックポイントを確立

### 2. 実装フェーズ

分析とビジュアライゼーションを開発します。

実装アプローチ：
- データ探索から始める
- 段階的に構築する
- 仮定を検証する
- 再利用可能なコンポーネントを作成する
- パフォーマンスを最適化する
- セルフサービス向けに設計する
- 徹底的にドキュメント化する
- エッジケースをテストする

分析パターン：
- まずデータ品質をプロファイリング
- ベースクエリを作成
- 計算レイヤーを構築
- ビジュアライゼーションを開発
- インタラクティビティを追加
- フィルターを実装
- ドキュメントを作成
- 更新をスケジュール
