# 品質・セキュリティ サブエージェント

品質・セキュリティサブエージェントは、コードエクセレンスとシステム保護の守護者です。これらのスペシャリストは、アプリケーションが堅牢で安全で高パフォーマンスかつアクセシブルであることを確保します。包括的なテスト戦略からセキュリティ監査、パフォーマンス最適化からコンプライアンス強制まで、最高水準の品質とセキュリティを満たすソフトウェアの構築を支援します。

## 品質・セキュリティサブエージェントを使うタイミング

以下の場合にこれらのサブエージェントを使用してください：
- **包括的なテスト**戦略と自動化の実装
- **脆弱性と脅威**からアプリケーションを保護
- **速度と効率**のためのパフォーマンス最適化
- **すべてのユーザー**のためのアクセシビリティの確保
- **コード品質**のレビューと標準の強制
- **複雑な問題**の体系的なデバッグ
- **規制**のコンプライアンス達成
- **カオスエンジニアリング**によるシステム耐障害性テスト

## 利用可能なサブエージェント

### [**accessibility-tester**](accessibility-tester.md) - アクセシビリティコンプライアンスエキスパート
すべての人のためにアプリケーションが動作することを確保するアクセシビリティスペシャリスト。WCAGガイドライン、スクリーンリーダーとの互換性、インクルーシブデザインを習得。機能性を損なわずにアプリケーションをアクセシブルにします。

**使用するとき：** アクセシビリティ機能の実装、WCAGコンプライアンスの監査、支援技術でのテスト、アクセシビリティ問題の修正、またはインクルーシブインターフェースの設計時。

### [**ad-security-reviewer**](ad-security-reviewer.md) - Active Directoryセキュリティポスチャーと攻撃面アナリスト
ディレクトリ設定、管理モデル、信頼境界をレビューするActive Directoryセキュリティスペシャリスト。設定ミス、過剰な権限、レガシープロトコル、一般的な攻撃パスへのAD強化に注力。

**使用するとき：** ADセキュリティポスチャーのレビュー、特権グループと委任の評価、GPOと認証設定の強化、またはラテラルムーブメントとドメイン侵害リスクを削減する強化計画時。

### [**architect-reviewer**](architect-reviewer.md) - アーキテクチャレビュースペシャリスト
スケーラビリティ、保守性、ベストプラクティスのためにシステム設計を評価するアーキテクチャエキスパート。アーキテクチャリスクを特定し改善を提案。長期的なシステムの健全性を確保します。

**使用するとき：** アーキテクチャ設計のレビュー、技術的決定の評価、アーキテクチャ負債の特定、リファクタリング計画、またはシステム設計の検証時。

### [**chaos-engineer**](chaos-engineer.md) - システム耐障害性テストエキスパート
弱点を発見するためにカオスエンジニアリングを使用する耐障害性スペシャリスト。障害注入、ゲームデイ、カオス実験を習得。制御されたカオスを通じてシステムの信頼性に対する信頼を構築します。

**使用するとき：** システム耐障害性テスト、カオスエンジニアリングの実装、障害シナリオの計画、フォールトトレランスの改善、または障害復旧の検証時。

### [**code-reviewer**](code-reviewer.md) - コード品質の守護者
徹底的なコードレビューを実施するコード品質エキスパート。ベストプラクティス、デザインパターン、コードスメルを習得。コードがクリーンで保守可能でチーム標準に従っていることを確保します。

**使用するとき：** プルリクエストのレビュー、コード標準の確立、技術的負債の特定、開発者のメンタリング、またはコード品質の改善時。

### [**compliance-auditor**](compliance-auditor.md) - 規制コンプライアンスエキスパート
規制と標準への準拠を確保するコンプライアンススペシャリスト。GDPR、HIPAA、SOC2、業界固有の要件を習得。複雑なコンプライアンスランドスケープを専門的に対応します。

**使用するとき：** 規制コンプライアンスの達成、データプライバシーの実装、監査準備、コンプライアンスのドキュメント化、または規制の理解時。

### [**debugger**](debugger.md) - 高度なデバッグスペシャリスト
最も複雑な問題を解決するデバッグエキスパート。言語とプラットフォームにわたるデバッグツール、技術、方法論を習得。他の人が諦めるところで根本原因を見つけます。

**使用するとき：** 複雑な問題のデバッグ、メモリリークの分析、競合状態の調査、アプリケーションのプロファイリング、または断続的なバグの解決時。

### [**error-detective**](error-detective.md) - エラー分析・解決エキスパート
捉えにくいバグを追跡するエラー調査スペシャリスト。ログ分析、エラーパターン、体系的なデバッグに精通。暗号的なエラーを実行可能な解決策に変えます。

**使用するとき：** 本番エラーの調査、エラーパターンの分析、エラー追跡のセットアップ、エラーハンドリングの改善、または分散システムのデバッグ時。

### [**penetration-tester**](penetration-tester.md) - 倫理的ハッキングスペシャリスト
脆弱性を見つけるために攻撃をシミュレートするセキュリティエキスパート。OWASP Top 10、ペネトレーションテストツール、エクスプロイト技術を習得。攻撃者のように考えてプロのように防御します。

**使用するとき：** セキュリティ評価の実施、脆弱性のテスト、セキュリティ修正の検証、セキュリティテストの実装、または外部監査の準備時。

### [**performance-engineer**](performance-engineer.md) - パフォーマンス最適化エキスパート
アプリケーションを超高速にするパフォーマンススペシャリスト。プロファイリング、最適化技術、パフォーマンステストを習得。ボトルネックを排除しリソース使用を最適化します。

**使用するとき：** アプリケーションパフォーマンスの最適化、負荷テストの実施、ボトルネックの分析、レスポンスタイムの改善、またはリソース消費の削減時。

### [**powershell-security-hardening**](powershell-security-hardening.md) - PowerShell駆動のセキュリティベースラインスペシャリスト
Windowsサーバー、ワークステーション、自動化ツールを強化するセキュリティ重視のPowerShellエキスパート。セキュアなリモーティング、制約付きエンドポイント、資格情報衛生、ロギング、スクリプトのセキュリティベースラインへの整合に特化。

**使用するとき：** Windowsホストの強化、PowerShellリモーティングの保護、スクリプトとスケジュールタスクのロックダウン、またはインフラ自動化をセキュリティベースラインとコンプライアンス要件に整合時。

### [**qa-expert**](qa-expert.md) - テスト自動化スペシャリスト
包括的なテスト戦略を設計するQAマスター。テスト自動化、フレームワーク、方法論のエキスパート。体系的なテストアプローチを通じて品質を確保します。

**使用するとき：** テスト自動化のセットアップ、テスト戦略の設計、CI/CDテストの実装、テストカバレッジの改善、またはQAプロセスの確立時。

### [**security-auditor**](security-auditor.md) - セキュリティ脆弱性エキスパート
徹底的なセキュリティ監査を実施するセキュリティスペシャリスト。脆弱性評価、セキュリティのベストプラクティス、修正戦略を習得。進化する脅威からアプリケーションを保護します。

**使用するとき：** アプリケーションセキュリティの監査、セキュリティのベストプラクティスの実装、脆弱性の修正、セキュアアーキテクチャの設計、またはチームへのセキュリティトレーニング時。

### [**test-automator**](test-automator.md) - テスト自動化フレームワークエキスパート
堅牢なテストフレームワークを構築する自動化スペシャリスト。様々なテストツール、パターン、戦略のエキスパート。保守可能で信頼性の高い自動テストスイートを作成します。

**使用するとき：** テストフレームワークの構築、テストケースの自動化、CI/CDへのテスト統合、テスト信頼性の改善、またはテスト自動化のスケーリング時。

## はじめに

1. アプリケーションの**品質懸念事項を特定**する
2. ニーズに**適切なスペシャリストを選択**する
3. アプリケーションコンテキストと既存の問題を**共有**する
4. 分析のための**関連コードとログを共有**する
5. 推奨される改善を**体系的に実装**する

## ベストプラクティス

- **シフトレフト：** 開発の早い段階で問題を発見
- **繰り返しを自動化：** 手動テストはスケールしない
- **全体を通じてセキュリティ：** セキュリティは後付けではない
- **パフォーマンスは重要：** ユーザーは高速なアプリを期待
- **アクセシビリティを含める：** すべてのユーザーのために設計
- **継続的にテスト：** 品質は継続的なもの
- **本番をモニタリング：** 実際の使用から学ぶ
- **発見事項をドキュメント：** チームと知識を共有

品質・セキュリティスペシャリストを選んで、より良いソフトウェアを構築しましょう！
