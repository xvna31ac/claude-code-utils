# philosophy

## 何のためのスキルか

SKILL.md の品質を「評価 → 承認 → 実装 → 再評価」のサイクルで実測値として改善するスキル。
グレードは目標ではなく、改善の結果として得られる指標。

## 評価の優先順序

動的eval（実際に動くか）が最優先。フォーマット準拠（発火・読み込みに影響）が次点。
静的スコアは改善方向の指針として使う。三者が矛盾する場合は動的evalを優先する。

## 改善の根拠は哲学にある

改善案の方向性は、スキルの核心哲学（philosophy.md）を起点に立案する。
矛盾が見つかったとき「どちらのファイルに書かれているか」は根拠にならない。
「どちらが核心哲学に沿っているか」で判断し、SKILL.md 本文であっても修正対象となる。

## 哲学の確立は一度きり

philosophy.md は評価サイクルの開始時に確立し、以後の評価で再度ヒアリングしない。
矛盾がなければ自動保存、軽微な問題なら軽量確認、哲学的不整合があれば3問ヒアリングで再合成する。
