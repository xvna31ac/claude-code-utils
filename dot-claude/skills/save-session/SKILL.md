# Save Session スキル

セッションの内容を次回以降の自分や他のAIエージェントが引き継げるよう、
構造化されたMarkdownファイルとして保存する。

---

## Step 1: 保存先の決定

以下の優先順位で保存先ディレクトリを決定する。

### 優先順位

1. **Gitブランチ名**

   ```bash
   git rev-parse --abbrev-ref HEAD 2>/dev/null
   ```

   ブランチ名が取れた場合: `tasks/<branch-name>/`

2. **チケットID（ブランチ名または会話中から検出）**
   - パターン: `[A-Z]+-\d+`（例: `LINEAR-123`, `JIRA-456`）、`#\d+`
   - 検出できた場合: `tasks/<ticket-id>/`

3. **会話からトピックを推定**
   - セッション全体のテーマを2〜4単語のkebab-caseで表現
   - 例: `setup-design-system`, `refactor-auth`, `add-payment-feature`
   - 保存先: `tasks/<inferred-topic>/`

### 注意

- `tasks/` はカレントディレクトリ（プロジェクトルート）直下に作成する
- 既存の `tasks/<dir>/session-*.md` があれば確認し、同じセッションの継続なら同ディレクトリを使う

---

## Step 2: セッションIDの取得

Claude Code のセッションIDを取得する。

```bash
# カレントディレクトリに一致する最新セッションを探す
cat ~/.claude/sessions/*.json 2>/dev/null \
  | grep -o '"sessionId":"[^"]*"' \
  | tail -1 \
  | grep -o '[0-9a-f-]\{36\}'
```

取得できない場合は `unknown` とする。

---

## Step 3: ファイル名の決定

```
session-YYYYMMDD-HHMMSS.md
```

タイムスタンプは現在の日時を使用する（`date +%Y%m%d-%H%M%S`）。
衝突しない・時系列順に並ぶ・複数セッション共存できる。

---

## Step 4: 内容の抽出と生成

会話履歴全体を振り返り、以下の構造で記述する。

```markdown
# セッション引継ぎ

**日時**: YYYY-MM-DD HH:MM
**セッションID**: <claude-session-id>
**ブランチ/チケット**: <取得できた場合のみ記載>

---

## プロジェクト概要

<このプロジェクトが何で、何を目指しているかを2〜4文で>

---

## このセッションでやったこと

<箇条書きで。「何をした」ではなく「何が変わったか・決まったか」を重視>

---

## 重要な決定事項

<今後のコードや設計に影響する判断を箇条書きで。理由も一言添える>

---

## これから

### 直近のネクストアクション
<次のセッションで最初にやるべきこと。具体的なタスクレベルで>

### 未決定事項
<検討中・保留中のもの。なければ省略>

---

## 現在のファイル構成

<作業ディレクトリの現状。新規作成・変更があったファイルを中心に>
```

### 記述のガイドライン

- **プロジェクト概要**はセッションをまたいで読む人向け。毎回同じ内容でよい
- **やったこと**は「ファイルを作った」より「○○の思想を整理してdocs/に反映した」のように成果を書く
- **決定事項**は次のAIが迷わないよう、判断の根拠も残す
- **ネクストアクション**は「技術選定」より「React + Vite の技術選定」のように具体的に
- ファイル構成はtreeコマンドで取得して整形してよい

---

## Step 5: ファイルの保存と報告

1. ディレクトリが存在しなければ作成
2. ファイルを保存
3. ユーザーに以下を報告:
   - 保存先パス
   - 保存先の決定理由（ブランチ名から / トピック推定から 等）
   - 既存のsessionファイルがある場合はその旨も
