# Load Session スキル

過去セッションを読み込み、現在の状況と「セッションをまたいで埋もれた事項」を整理して報告する。

---

## Step 1: 対象フォルダの特定

以下の優先順位で `tasks/` 以下のどのフォルダを読むか決める。

1. **Gitブランチ名**

   ```bash
   git rev-parse --abbrev-ref HEAD 2>/dev/null
   ```

   → `tasks/<branch-name>/` が存在すればそこを使う

2. **チケットID**（ブランチ名または会話中から `[A-Z]+-\d+` / `#\d+` を検出）
   → `tasks/<ticket-id>/` が存在すればそこを使う

3. **tasks/ 以下のフォルダ一覧を提示してユーザーに選ばせる**

   ```bash
   ls tasks/
   ```

   複数ある場合は一覧を出して確認する。

---

## Step 2: セッションファイルの読み込み

対象フォルダ内の `session-*.md` を**タイムスタンプ順（古い順）に全件**読み込む。

```bash
ls tasks/<folder>/session-*.md | sort
```

件数が多い場合（10件超）は古いものから要約しつつ、直近5件は全文読む。

---

## Step 3: 埋もれた事項の検出

全セッションを時系列で比較し、以下を検出する。

### 検出対象

**A. 手つかずのネクストアクション**

- あるセッションの「直近のネクストアクション」に記載された
- 以降のセッションの「やったこと」に対応する記述がない

**B. 放置された未決定事項**

- あるセッションの「未決定事項」に記載された
- 以降のセッションで言及・解決された形跡がない

**C. 中断されたトピック**

- 「やったこと」や「決定事項」に現れたが、その後のセッションで深掘りされていない話題
- 特に「〜は別途検討」「〜は後で」のような保留ワードと共に登場したもの

### 判断の注意点

- 完了しているかどうかは「やったこと」「決定事項」との照合で判断する
- 意図的にスコープ外になったものと、単に忘れられたものを区別して報告する
- 確信が持てない場合は「要確認」として分類する

---

## Step 4: 報告

以下の構造で報告する。

```markdown
## セッション読み込み完了

**対象**: tasks/<folder>/ （全N件）
**期間**: YYYY-MM-DD 〜 YYYY-MM-DD

---

### 現在の状況
<最新セッションのプロジェクト概要・状態を2〜3文で>

---

### 前回のネクストアクション
<最新セッションの「直近のネクストアクション」をそのまま列挙>

---

### ⚠️ 埋もれていた事項
<検出された手つかず・放置・中断トピックを列挙>
<各項目に「いつのセッションで登場したか」を添える>
<完了済みと判断できるものは除外する>

埋もれた事項がなければ「特になし」と記載する。

---

### 未決定事項（継続中）
<最新セッション時点でまだ未決定のもの>

---

このセッションで何から始めますか？
```

### 報告のガイドライン

- 埋もれた事項は責めるトーンにしない。「見落としていた可能性があります」程度に
- 項目が多い場合は重要度が高そうなものを上に出す
- 「要確認」は別グループにまとめて最後に置く
