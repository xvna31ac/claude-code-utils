# philosophy

## 何のためのスキルか

`.claude/rules/` モジュラールールシステムの**最適化**が目的。
単純なルール抽出ではなく、rules/ がなければ構築し、あれば改善・追加・整理する。

## 対象の見方

プロジェクト資材（設定ファイル・コード等）はプロジェクトを理解するための**読み取り元**。
CLAUDE.md と rules/ は**最適化対象**であり、抽出元ではない。

モード判定の軸は rules/ の有無のみ。CLAUDE.md の有無は関係しない。

## 何を rules/ に入れるか

プロジェクト固有の振る舞いルールのみ。
判定基準は「このルールがなければ Claude が別の動作をするか」。
Claude がデフォルトで行うこと・用語定義・知識は対象外。
