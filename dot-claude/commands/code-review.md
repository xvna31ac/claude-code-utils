# コードレビュー

コミットされていない変更のセキュリティと品質の包括的レビュー:

1. 変更ファイルを取得する: `git diff --name-only HEAD`

2. 変更された各ファイルについて以下を確認する:

**セキュリティの問題（CRITICAL）:**

- ハードコードされた認証情報・APIキー・トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- インプット検証の欠如
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質（HIGH）:**

- 50行超の関数
- 800行超のファイル
- ネストの深さが4レベル超
- エラーハンドリングの欠如
- console.log文
- TODO/FIXMEコメント
- 公開APIのJSDocなし

**ベストプラクティス（MEDIUM）:**

- ミュータブル操作パターン（イミュータブルを優先すること）
- コード/コメント内の絵文字使用
- 新しいコードのテスト不足
- アクセシビリティの問題（a11y）

1. 以下を含むレポートを生成する:
   - 重大度: CRITICAL・HIGH・MEDIUM・LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 提案する修正

2. CRITICALまたはHIGHの問題が見つかった場合はコミットをブロックする

セキュリティ脆弱性のあるコードは絶対に承認しない！
