# ドメインスキルテンプレート

workflow-domain-skill-generator が生成するドメインスキルの完全テンプレート。skill-creator 準拠の構造化JSON出力形式。

---

## 構造化JSON出力テンプレート（skill-creator 準拠）

workflow-domain-skill-generator はこの形式でJSONを返却する:

```json
{
  "skillName": "{{skill-name}}",
  "frontmatter": {
    "name": "{{skill-name}}",
    "description": "{{description}}"
  },
  "body": "{{SKILL.md本体のmarkdownテキスト（frontmatter以降）}}",
  "references": [
    {
      "filename": "references/{{reference-file}}.md",
      "content": "{{referencesファイルの内容}}"
    }
  ],
  "optimizationReport": {
    "issuesFound": [
      {
        "pattern": "BP-XXX",
        "severity": "P1|P2|P3",
        "location": "セクション名",
        "transform": "適用した変換の説明"
      }
    ],
    "lineCount": 0,
    "sizeCategory": "small|medium|large",
    "principlesApplied": ["1: コンテキスト効率", "4: 測定可能性"]
  },
  "metadata": {
    "tags": ["{{tag1}}", "{{tag2}}"],
    "typicalUse": "{{典型的な使用場面}}",
    "sections": ["コンテキスト", "中核概念", "プロセス"],
    "keyReferences": ["{{ref1}}"]
  }
}
```

- {{if-then基準1}}

{{具体的な手順}}

## 出力形式

```{{format}}

## BP-001〜008 適用チェックリスト

生成前に以下を確認:

| パターン | チェック項目 |
|---------|------------|
| BP-001 | 否定形指示を肯定形に変換済みか |
| BP-002 | 曖昧な表現を測定可能な基準に置換済みか |
| BP-003 | プロセスセクションに出力形式を追加済みか |
| BP-004 | 標準セクション順序（コンテキスト→概念→プロセス→出力→チェックリスト→参照）に従っているか |
| BP-005 | 前提条件が明示されているか |
| BP-006 | 複雑な指示をチェックポイント付きステップに分解済みか |
| BP-007 | 正常系・エッジケース・エラーケースの例が含まれているか |
| BP-008 | 不確実時のエスカレーション基準が明示されているか |

## description 生成ガイドライン

```
形式: {対象}を{基準}で{動詞}。{使用場面}時に使用。
文体: 三人称・動詞始まり
文字数: 最大1024文字
```

**例**:
```
コードレビューを品質基準とセキュリティパターンで評価。PR作成前・マージ承認時に使用。
要件仕様を構造化してスコープ・依存関係・規模を判定。設計フェーズ開始前に使用。
```

---

## サイズ基準

| サイズ | 行数 | 対処 |
|--------|------|------|
| small | 80行未満 | 参照ファイル不要 |
| medium | 80-250行 | 具体例は references/ に移動可 |
| large | 250行以上 | 参照データを references/ に抽出。本体は250行以内 |

500行制限: SKILL.md本体は500行以内に収める。
