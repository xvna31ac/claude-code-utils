## 目的

フロントエンド実装のWAI-ARIA準拠・WCAG 2.1 AAレベル適合・キーボードアクセシビリティを静的解析し、
障害を持つユーザーが利用可能なインターフェースになっているかを評価してレポートを生成する。

## 実行プロセス

### Step 1: コンテキストと対象範囲の把握

**実行内容**:
- 対象のUIコンポーネント・ページ構成を確認する
- 使用しているUIフレームワーク（React, Vue, Angular等）と関連ライブラリを確認する
- 既存のアクセシビリティ設定・テストの有無を確認する
- 優先的に確認すべき重要ユーザーフロー（フォーム、ナビゲーション等）を特定する

**専門性の要件**: フロントエンド開発、アクセシビリティ標準

**期待成果物**: 分析対象コンポーネントと重要ユーザーフローの一覧

**完了基準**: 全ての対象コンポーネントとユーザーフローが識別されている

### Step 2: セマンティックHTML評価

**実行内容**:
- HTML要素の意味論的な適切性を確認する
  - 見出し階層（h1→h2→h3）の正しい使用
  - ランドマーク要素（header, nav, main, footer, aside）の適切な配置
  - リスト要素（ul/ol/li）の適切な使用
  - 表（table/thead/tbody/th）の適切なマークアップ
- divとspanの過剰使用（DivSoup）を特定する
- インタラクティブ要素（ボタン、リンク）が適切なHTML要素で実装されているかを確認する
- フォーム要素のlabel関連付けを確認する（for属性またはwrap）

**専門性の要件**: HTMLセマンティクス、WAI-ARIA

**期待成果物**: セマンティックHTML問題一覧

**完了基準**: 非セマンティックな実装が全て特定され、改善HTMLが提示されている

### Step 3: WAI-ARIA 実装評価

**実行内容**:
- ARIA属性の正しい使用を確認する
  - `role`属性の適切な使用（暗黙のロールとの重複を除く）
  - `aria-label`/`aria-labelledby`/`aria-describedby`の適切な使用
  - `aria-expanded`/`aria-checked`/`aria-selected`等の状態属性の動的更新
  - `aria-live`/`aria-atomic`の適切な設定（動的コンテンツ更新）
  - `aria-hidden`による支援技術からの適切な非表示
- ARIAアンチパターンを検出する
  - `role="presentation"`/`aria-hidden="true"`のフォーカス可能要素への誤用
  - 存在しない子要素を必要とするロールの単独使用
  - 不必要なARIA属性の冗長使用
- カスタムコンポーネント（ドロップダウン、モーダル、タブ等）のARIAパターン準拠を確認する

**専門性の要件**: WAI-ARIA Authoring Practices、支援技術

**期待成果物**: ARIA実装問題一覧（ARIA in HTML規格参照付き）

**完了基準**: 全てのARIA違反が特定され、正しい実装例が提示されている

### Step 4: キーボードアクセシビリティ評価

**実行内容**:
- フォーカス管理を確認する
  - 全てのインタラクティブ要素がTab/Shift+Tabでフォーカス可能かを確認する
  - フォーカス順序（tabindex）が論理的なDOMフローに従っているかを確認する
  - `tabindex="-1"`と`tabindex="0"`以外の正の値の使用を検出する（アンチパターン）
  - モーダル/ダイアログのフォーカストラップが適切に実装されているかを確認する
  - ルーティング変更時のフォーカス管理を確認する
- キーボード操作パターンを確認する
  - ボタン: Enter/Spaceで操作可能か
  - リンク: Enterで遷移可能か
  - チェックボックス/ラジオ: Spaceで切り替え可能か
  - ドロップダウン: 矢印キーでナビゲーション可能か
  - ESCキーでモーダル/ポップオーバーを閉じられるか
- フォーカスインジケーターが視覚的に確認できるかを確認する
  - `outline: none`/`outline: 0`の不適切な使用を検出する

**専門性の要件**: キーボードアクセシビリティ、フォーカス管理

**期待成果物**: キーボードアクセシビリティ問題一覧

**完了基準**: キーボード操作不能な全インタラクティブ要素が特定されている

### Step 5: コントラスト比・視覚的アクセシビリティ評価

**実行内容**:
- テキストとその背景のコントラスト比をWCAG基準で評価する
  - 通常テキスト: 4.5:1以上（WCAG AA）
  - 大テキスト（18pt/14pt Bold以上）: 3:1以上（WCAG AA）
  - UI コンポーネント境界: 3:1以上（WCAG AA）
- カラーのみで情報を伝えているケースを特定する（色盲ユーザーへの対応）
  - エラー表示が赤色のみかどうかを確認する
  - 必須フィールドの識別が色のみかどうかを確認する
- テキストの最小フォントサイズ（推奨: 12px以上）を確認する
- アニメーション・点滅コンテンツの `prefers-reduced-motion` 対応を確認する

**専門性の要件**: WCAG 2.1、色彩アクセシビリティ

**期待成果物**: コントラスト比問題一覧（具体的な比率値付き）

**完了基準**: WCAG AA基準を下回る全てのコントラスト比問題が特定されている

### Step 6: スクリーンリーダー互換性評価

**実行内容**:
- 画像の代替テキストを確認する
  - 意味のある画像: 適切な`alt`テキストがあるかを確認する
  - 装飾的な画像: `alt=""`が設定されているかを確認する
  - SVGアイコン: `aria-label`または`title`があるかを確認する
  - 背景画像で意味のあるもの: テキスト代替があるかを確認する
- フォームの案内情報を確認する
  - エラーメッセージがフォームフィールドに紐付いているかを確認する（aria-errormessage/aria-describedby）
  - 必須フィールドが明示的に示されているかを確認する（aria-required）
  - ヒントテキストがフィールドに紐付いているかを確認する
- 動的コンテンツ更新のアナウンスを確認する（aria-live regions）
- リンクテキストの意味が文脈なしで理解できるかを確認する（「ここをクリック」等の禁止）

**専門性の要件**: スクリーンリーダー動作原理、支援技術

**期待成果物**: スクリーンリーダー互換性問題一覧

**完了基準**: スクリーンリーダーユーザーが操作できない全箇所が特定されている

### Step 7: アクセシビリティチェックレポート生成

**実行内容**:
- 発見した問題をWCAG成功基準（SC番号）とレベル（A/AA/AAA）で分類する
- 問題を重大度（Critical/Major/Minor）で優先順位付けする
- 各問題に対して修正HTMLコードを提示する
- WCAG 2.1 AA準拠チェックリストを完成させる
- 総合的なアクセシビリティスコアを算出する

**専門性の要件**: WCAG 2.1、アクセシビリティ監査

**期待成果物**: アクセシビリティチェックレポート

**完了基準**: 全ての問題がWCAG成功基準に紐付けられ、修正コードが提示されている

## 品質基準

このSkillの完了条件:
- [ ] セマンティックHTML評価が完了している
- [ ] WAI-ARIA実装の正確性が検証されている
- [ ] 全てのインタラクティブ要素のキーボードアクセシビリティが確認されている
- [ ] WCAG AA基準のコントラスト比が評価されている
- [ ] スクリーンリーダー利用者向けの代替テキストが確認されている
- [ ] 全問題にWCAG成功基準番号が付与されている

## 成果物

- **アクセシビリティチェックレポート**: `.workflow/reports/accessibility.md`
  - 総合アクセシビリティスコア
  - WCAG 2.1 AA準拠チェックリスト
  - Critical問題一覧（キーボード操作不能、スクリーンリーダー不可等）
  - Major問題一覧（コントラスト比不足、ARIAエラー等）
  - Minor問題一覧（推奨改善事項）
  - 修正後確認チェックリスト

## WCAG 2.1 AA 主要チェックリスト

| 原則 | 主なSC | チェック内容 |
|------|--------|------------|
| 知覚可能 | 1.1.1 | 非テキストコンテンツの代替テキスト |
| 知覚可能 | 1.3.1 | 情報と関係性の保持 |
| 知覚可能 | 1.4.3 | コントラスト比 4.5:1 |
| 操作可能 | 2.1.1 | キーボード操作可能 |
| 操作可能 | 2.4.3 | フォーカス順序 |
| 操作可能 | 2.4.7 | フォーカスの可視性 |
| 理解可能 | 3.3.1 | エラーの識別 |
| 堅牢性 | 4.1.2 | 名前・役割・値 |
